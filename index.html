System Snapshot Webserver.

Collect the computer and system info at a given time in order to track issues or problems. You can share the generated report to help you detect your computer problem at a given moment. Works on Windows / Linux OS.

What kind of Reports

  1. System Info
  2. Hardware
  3. Networks
  4. PnP Devices
  5. Processes
  6. Services
  7. Benchmark
  8. Installed programs
  9. Special files

How to use

Run program (.exe) as Administrator. Wait util report gets generated. Open or send the report to someone who can read it and analyse or problem.

Note: Program can be executed with '-s', '/s' or '--silent' argument to skip GUI. Report will be generated and the html file will show on explorer after completion.

Example: "SystemSnapshotWebServer.exe -s"

Arguments

Null mode

Arguments: '-n', '/n' or '--null' Description: Only generate the report without showing or doing anything. Good for scripts or tasks. Default: False. Example: "SystemSnapshotWebServer.exe --null" Silent mode

Arguments: '-s', '/s' or '--silent' Description: Run program without showing the GUI. After the report is generated that will be shown on explorer after completion. Default: False. Example: "SystemSnapshotWebServer.exe -s" Open report on completion

Arguments: '-o', '/o' or '--open-report' Description: After the report is generated that will be opened automatically in the default browser. Default: False. Example: "SystemSnapshotWebServer.exe -o" Define the file path destination

Arguments: '-f', '/f' or '--filename' Description: Set the path and/or filename for the generated report file. If the passed path is a directory the default filename will be appended to it. Absolute or relative paths are allowed. Default: Desktop/SystemSnapshotWebServer_date-time.html. Examples: "SystemSnapshotWebServer.exe -f ." - Use a dot (.) to set the path as same as the executable path or working dir and preserve default filename. "SystemSnapshotWebServer.exe -f="C:\my folder"" - (See note 2) "SystemSnapshotWebServer.exe -f:/home/snapshots/report.html" - (See note 3 and 7) "SystemSnapshotWebServer.exe -f mynewreport.html" - (See note 4) "SystemSnapshotWebServer.exe -f D:\report1" - (See note 5) Notes: Path must be writable! If a path from a existing directory is passed, filename will be set as default and saved on this directory. ie: -f D:\ If the passed path is not a directory then it will try to save the file in this path. ie: -f "D:\myreport.html" Absolute and relative paths are allowed. Relative paths will start from the working directory (from executable path). ie: -f report.html The .html extension will be appended if not present in the filename. ie: -f report If you give a existing file path, that file will be overwritten without warning. ALWAYS use -f:path OR -f=path under a Unix system, because slash (/) is interpreted as a argument. ie: "-f /home" = ERROR | "-f:/home" = OK Max tasks to use on reports generation

Arguments: '-t', '/t' or '--max-tasks' value:int Description: Sets the maximum number of concurrent tasks enabled to generate the reports. If it is -1, there is no limit on the number of concurrently running operations (Default). If it is 1 it will run in a single thread, best used with single core CPUs or for debuging. Default: -1. Examples "SystemSnapshotWebServer.exe -t 0" - Use 0 or lower to use no limit, best value will be choose by .NET Framework. (Default) "SystemSnapshotWebServer.exe -t 1" - Allow only 1 maximum task (single thread). "SystemSnapshotWebServer.exe -t 4" - Allow only 4 maximum tasks. Note: See ParallelOptions > MaxDegreeOfParallelism for more information about it. Listen server

Arguments: '--listen-server' or '/listen-server' ip:port [update-interval:int] Description: Create a local web server and display live information about this machine on the web. Default: *:8080 30. Examples: "SystemSnapshotWebServer.exe --listen-server *" - Listen the server on any IP in the default port with 30 seconds of update interval. "SystemSnapshotWebServer.exe --listen-server localhost:803" - Listen the server on the 127.0.0.1 IP in the 803 port with 30 seconds of update interval. "SystemSnapshotWebServer.exe --listen-server * 20" - Listen the server on any IP in the default port with 20 seconds of update interval. "SystemSnapshotWebServer.exe --listen-server *:600 30" - Listen the server on any IP in the 600 port with 30 seconds of update interval. "SystemSnapshotWebServer.exe --listen-server *:600 30 --max-tasks 1" - Same as above but set the program to work in a single thread to keep low CPU usage. Notes: Chosen TPC port must be free, if not server will not start and a error is shown. A report will be generated and kept every "Update interval" in seconds. When using this option the reports will not be saved anymore in files and exists only in memory. To keep the CPU usage to the minimum i recommend using this options with '--max-tasks' option set to 1 and work in Single thread.

"SystemSnapshotWebServer.exe --null -o" - Generate and open the report in the default browser without showing the GUI. "SystemSnapshotWebServer.exe -s -o" - Generate, show and open the report in the explorer and the default browser without showing the GUI. "SystemSnapshotWebServer.exe --null -o -f D:\snapshots" - Generate and open the report in default browser without showing the GUI. File will be saved on folder D:\snapshots if exists, otherwise a snapshots.html will be created. "SystemSnapshotWebServer.exe --null --max-tasks 1 --listen-server *:8080 20" - Listen a server on any IP address on the port 8080 and update the report every 20 seconds. Also set to 1 task in order to save the CPU and keep at minimum usage when generating a new report on background. No GUI.
